# 02 技术架构与接入点

## 1. 与现有 ZENdian 架构的关系

Homepage 应作为一个标准模块接入现有 ModuleManager。

现有模块形态大致为：

```ts
class SomeModule {
  id = "some-module";
  name = "模块名称";

  async load(ctx) {
    // 注册事件、应用 class、读取设置
  }

  unload() {
    // 清理事件、class、timer、observer
  }

  onSettingsChanged(changedKeys, settings) {
    // 响应设置变化
  }
}
```

Homepage 也应遵循该结构：

```ts
class HomepageModule {
  id = "homepage";
  name = "主页控制台";

  async load(ctx: ModuleContext) {}
  unload() {}
  onSettingsChanged(changedKeys, settings) {}
}
```

```text
│   ├── HomepageView.ts
│   └── HomepageModule.ts
└── main.ts
```

## 3. 插件入口改动

在 `main.ts` 中：

```ts
import { HomepageModule } from "./homepage/HomepageModule";

this.moduleManager.register(new HomepageModule(this));
```

如果当前 ModuleManager 的 module 构造不接收 plugin 实例，则建议给 HomepageModule 构造函数传入 plugin，因为 `registerView()`、`addCommand()`、`registerEvent()` 是 Obsidian Plugin 实例方法，单靠 ctx 不够方便。

## 4. DEFAULT_SETTINGS 改动

在默认设置中新增：

```ts
homepage: {
  enabled: true,
  replaceEmptyTab: false,
  openOnStartup: false,
  focusSearchOnOpen: true,
  showStats: true,
  showRecentFiles: true,
  recentFileLimit: 10,
  showQuickActions: true,
  enableCreateFromSearch: true,
  defaultCreateFolder: ""
}
```

## 5. 设置迁移

你现有 `SettingsManager.migrate()` 已经支持“如果老配置是对象，则按 key 合并默认配置”的策略。因此只要把 `homepage` 放入 `DEFAULT_SETTINGS`，旧用户升级后一般能自动得到默认 homepage 配置。

## 6. 视图注册

HomepageModule 在 `load()` 中执行：

```ts
plugin.registerView(
  VIEW_TYPE_ZENDIAN_HOMEPAGE,
  leaf => new HomepageView(leaf, ctx)
);
```

## 7. 命令注册

HomepageModule 注册至少三个命令：

```ts
open-zendian-homepage
reset-zendian-homepage
refresh-zendian-homepage
```

MVP 可以先只实现第一个。

## 8. Empty tab 接管

监听：

```ts
app.workspace.on("active-leaf-change", handler)
```

判定：

```ts
leaf.getViewState().type === "empty"
```

替换：

```ts
await leaf.setViewState({
  type: VIEW_TYPE_ZENDIAN_HOMEPAGE,
  active: true
});
```

必须加保护：

- 首页功能关闭时不替换
- 设置关闭时不替换
- 当前 leaf 已是 homepage 时不替换
- 正在替换中时不重复替换

## 9. CSS 作用域

所有主页样式必须限定在：

```css
.workspace-leaf-content[data-type="zendian-homepage-view"]
```

不要污染：

```css
.workspace-leaf-content[data-type="empty"]
```

这样可以和现有 new tab 美化共存。

## 10. 与现有 newTab 的关系

当前 `newTab` 配置控制的是原生 empty tab 的视觉美化。Homepage 是更高阶的替代模式。

建议设置命名区分：

- `newTab`：原生空白页样式
- `homepage`：自定义主页视图与接管逻辑

当 `homepage.replaceEmptyTab = true` 时，原生 empty tab 很少出现，`newTab` 样式作为 fallback 保留。

## 11. 性能策略

MVP 不建立全量搜索索引，只在输入时对 `app.vault.getFiles()` 做轻量过滤。

后续 V2 再加入：

- 内存索引
- 防抖重建
- 最近访问权重
- metadataCache 事件聚合
